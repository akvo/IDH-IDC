## ADR-006: Scenario-Segment Multi-Selector in Step 5

- **Status**: Accepted
- **Context**: 
  Users need to compare the cost-effectiveness (ROI/Costs) of specific scenario-segment combinations side-by-side. The current implementation in `ImpactOfInvestmentCharts.js` uses separate Scenario and Segment dropdowns, which is inconsistent with Step 3's comparison pattern and limits the ability to compare "Segment A in Scenario 1" vs "Segment B in Scenario 2".

- **Decision**: 
  1. Replace `selectedScenarioKey` and `selectedSegmentId` state variables with a single `selectedScenarioSegments` array.
  2. Adopt the concatenated string format `${scenarioKey}-${segmentId}` as the unique identifier for each selection (aligning with `ChartIncomeGapAcrossScenario.js`).
  3. Implement a multi-select `Select` component with a maximum limit of 5 items.
  4. Refactor `roiData` and subsequent chart data hooks to map over the active selections, deriving display names like `[Scenario Name] - [Segment Name]`.

- **Alternatives Considered**: 
  - **Keeping Separate Selects**: Rejected because it only allows comparing all segments for one scenario or one segment across all scenarios, but not disparate combinations.
  - **Hierarchical/Tree Select**: Rejected to maintain consistency with the existing multi-select pattern used in Step 3.

- **Consequences**:
  - **Consistency**: Visualizations across Steps 3 and 5 will now share the same interaction model.
  - **Flexibility**: Users can perform highly specific comparisons between different strategic interventions.
  - **Complexity**: Requires a more sophisticated data mapping logic to handle multiple independent data sources (Scenario/Segment pairs).
